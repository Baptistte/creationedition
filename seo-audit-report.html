Problèmes critiques — Technique

C1 URLs du sitemap déclenchent des redirections 301 (slashes finaux manquants) Critique
sitemap.xml — toutes les <loc>

Netlify sert toutes les pages avec un slash final et redirige les URLs sans slash (HTTP 301). Or, le sitemap déclare toutes les URLs sans slash final. Résultat : chaque URL du sitemap (36 au total) déclenche une redirection avant que Googlebot atteigne la page finale. Cela consomme du crawl budget inutilement et dilue les signaux de lien transmis par la découverte via sitemap.

Action requise
Ajouter un / final à l'ensemble des 36 entrées <loc> du fichier sitemap.xml. Exemple : /services/creation-graphique-digitale/ au lieu de /services/creation-graphique-digitale.
C2 Balises canonical pointent vers la source de la redirection (sans slash final) Critique
services/creation-graphique-digitale/index.html · services/broderie-marquage-textile/index.html · services/broderie-personnalisee/index.html

Les balises <link rel="canonical"> de ces trois pages pointent vers des URLs sans slash final. Comme Netlify sert ces pages avec un slash, le canonical désigne l'URL source d'une redirection 301 — pas la destination finale. Google peut alors consolider le PageRank vers la mauvaise URL.

Action requise
Ajouter le slash final dans tous les attributs href des balises canonical. Ex : href="https://creationeditionbroderie.com/services/creation-graphique-digitale/"
C3 Pages de services dupliquées à deux chemins URL sans relation canonique Critique

Plusieurs pages existent à deux emplacements simultanément :

Chemin plat (sitemap)Chemin imbriqué (navigation)
/services/broderie-personnalisee//services/broderie-marquage-textile/broderie-personnalisee/
/services/flocage-marquage-textile//services/broderie-marquage-textile/flocage-marquage-textile/
/services/vetements-travail-personnalises//services/broderie-marquage-textile/vetements-travail-personnalises/
Action requise
Le chemin imbriqué (utilisé dans la navigation) devient le chemin canonique. Configurer une redirection 301 de chaque chemin plat vers le chemin imbriqué dans netlify.toml. Mettre à jour le sitemap pour utiliser les chemins imbriqués.

Problèmes de haute priorité — Technique

H1 Titres <title> au format question sur toutes les pages de services Haute

L'ensemble des pages de services utilise un format question comme titre HTML (<title>) :

PageTitre actuel

Ce format convient aux articles de blog mais nuit aux pages de services — il ne communique pas d'expertise et crée une incohérence avec les OG titles (qui utilisent déjà le format mot-clé + marque).

Format recommandé

Ex : Création Logo & Identité Visuelle Fréjus | Création Édition & Broderie
H2 Image LCP sans fetchpriority="high" ni preload — retard de chargement Haute
index.html

La première image identifiée (casquette_test.webp) possède l'attribut loading="lazy", ce qui retarde activement son chargement si elle est l'élément LCP. Aucune balise <link rel="preload"> n'est présente dans le <head> pour les images au-dessus de la ligne de flottaison.

Action requise
1. Identifier l'élément LCP via PageSpeed Insights. 2. Ajouter fetchpriority="high" sur cet élément <img>. 3. Ajouter <link rel="preload" as="image" href="..." fetchpriority="high"> dans le <head>. 4. Supprimer loading="lazy" de toute image dans le viewport initial.

Problèmes de priorité moyenne — Technique

M1 IntersectionObserver re-cache les éléments au scroll (risque CLS) Moyenne
index.html — script JS inline

Le JavaScript de la homepage re-cache les sections quand l'utilisateur remonte (opacity: 0; transform: translateY(20px) dans le bloc else de l'observer). Cela peut masquer du contenu lors du rendu Google et provoquer des décalages de mise en page (CLS).

Action requise
Supprimer le bloc else de l'IntersectionObserver. Appeler observer.unobserve(entry.target) après la première révélation de l'élément.
M2 Widget Elfsight sans hauteur réservée — risque CLS au chargement Moyenne

Le conteneur du widget Elfsight (avis clients) n'a pas de hauteur minimum réservée. Quand le script tiers se charge, il pousse le contenu vers le bas, générant un score CLS dégradé.

Action requise
Ajouter style="min-height: 300px" au <div class="elfsight-app-...">.
M3 pdf.js (1,4 MB) chargé globalement sur toutes les pages Moyenne

La librairie pdf.js est chargée en defer sur la homepage alors qu'aucun viewer PDF n'est visible. Cela représente ~1,4 MB de JS parsé inutilement sur chaque page, impactant le LCP et l'INP.

Action requise
Charger pdf.js uniquement via import() dynamique déclenché par l'interaction avec un élément PDF spécifique.

Améliorations du schema Homepage

H6 LocalBusiness sous-spécifié — devrait être ProfessionalService + WebSite manquant Haute

Le @type: "LocalBusiness" est valide mais trop générique. Pour une agence créative, ProfessionalService est le type approprié. De plus, le bloc WebSite est absent — il est nécessaire pour être éligible à la Sitelinks Search Box et améliorer la consolidation d'entité dans le Knowledge Graph.

Autres problèmes identifiés dans le schema homepage :

  • @id utilise l'URL de la page au lieu d'un fragment URI stable (#organization)
  • La propriété image devrait être un ImageObject avec url, width, height
  • La propriété logo est absente (distincte de image)
  • priceRange: "$$" est un format américain — utiliser "À partir de 390€"
  • Les services dans makesOffer manquent d'url, description et areaServed
  • sameAs n'inclut qu'un lien Google Maps — ajouter Facebook, Instagram, LinkedIn
  • Les URLs Facebook dans le schema ne correspondent pas aux URLs dans le footer (deux formats différents)
